# Metadata Directory - DB-4

**Purpose:** This directory stores metadata files related to database db-4, including pipeline execution logs, data quality reports, and schema metadata.

## Contents

- **pipeline_metadata.json**: Execution metadata from ETL/ELT pipeline runs
- Additional metadata files as needed

## Metadata Files

### pipeline_metadata.json

Contains:
- Pipeline execution timestamp
- Data sources processed
- Validation results
- Load results
- Data quality reports
- Pipeline status

## Usage

Metadata files are automatically generated by the ETL/ELT pipeline notebook in `../research/etl_elt_pipeline.ipynb`.

## Notes

- Metadata files are in JSON format for easy parsing and analysis
- Files are updated on each pipeline execution
- Historical metadata can be archived for tracking changes over time
